# Documentos do projeto

Este é um exemplo de documento que você pode salvar neste diretório. Use este espaço para reunir informação útil durante o desenvolvimento do projeto.

Sugestões de conteúdo:

- Planejamento: roadmap, requisitos, backlog
- Design: arquitetura, diagramas e decisões técnicas
- Operação: instruções de deploy, runbooks e configuração de ambientes
- Notas: decisões e justificativas

Boas práticas:

- Use nomes claros e consistentes para os arquivos (ex.: `design-api.md`, `deploy-postgres.md`).
- Preferencialmente mantenha a documentação em Markdown para facilitar leitura e diffs.
- Atualize a documentação junto com o código e descreva o porquê das decisões.
- Versione tudo com Git: commits pequenos e mensagens claras ajudam a rastrear mudanças.


Dica: documente não apenas o "o quê", mas também o "porquê" — isso facilita a integração de novos colaboradores.
